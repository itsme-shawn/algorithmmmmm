# Repository Guidelines

이 저장소는 알고리즘 문제 풀이(주로 BOJ 및 programmers) 아카이브입니다. 아래 내용은 이 코드베이스에서 AI 코딩 에이전트가 즉시 생산적으로 작업하도록 돕기 위한 구체적이고 실용적인 지침입니다.

당신(AI 코딩 에이전트)의 역할은 내가 푼 알고리즘 풀이에 대한 첨삭 / 더 나은 풀이파일 생성 / 복기용(리뷰) 마크다운 작성입니다.

## 프로젝트 구조
- 플랫폼 별로 폴더가 구분됨: `BOJ/1000~9999/<문제번호>/<문제번호>.py`, `BOJ/10000~/...`, `programmers/10000_99999/<문제번호>/<문제번호>.py`.
- 학습용 자료와 템플릿은 `알고리즘 기초`와 `알고리즘 기초/.template`, 기타 정리는 `기타`, 플랫폼별 추가 예시는 `codetree`.
- 언어는 파이썬이 주력이며, 일부 문제에 자바(.java)와 마크다운 해설(.md)이 섞여 있음. 새 파일을 만들 때는 주변 파일의 언어와 이름 규칙을 맞춰서 배치.

## 코딩 스타일 & 네이밍
- 파이썬: 4스페이스 들여쓰기, snake_case 변수명, 표준 라이브러리 위주 사용. 입력은 `sys.stdin.readline` 패턴을 유지하고, 불필요한 `print` 디버깅은 남기지 않음.
- 파일/폴더명은 문제 번호를 그대로 사용하고 확장자는 언어에 맞춤(`<문제번호>.<ext>`). 예: `BOJ/10000~/11286/11286.py`, `BOJ/10000~/11286/11286_2.java`.
- 주석과 설명은 알고리즘 의도와 예외 처리 위주로 짧게 작성. 한글 해설 시 기존 문서 톤과 일관성을 유지.

## 학습/풀이 진행 방식
- 각 문제는 단일 파일 풀이가 기본이며, 입력 예제는 필요 시 `in.txt`로 두고 리다이렉션으로 실행.
- `programmers` 스타일 함수는 `if __name__ == "__main__":` 블록을 활용해 간단한 로컬 검증을 추가했다가 제출 전 간결하게 정리.
- 시간복잡도와 메모리 사용을 문제 요구에 맞추어 점검(O(N log N) 등을 명시)하고, 스트림 입력 처리 흐름을 해치지 않는 범위에서만 리팩터링.

## 커밋 & PR 가이드
- 커밋 메시지 예시: `Add BOJ/<범위>/<문제번호> — <한 줄 요약>`, `Refactor <path> — <이유>`. 불필요한 파일 변경이나 포맷팅만의 변경은 묶지 않음.
- PR/설명 작성 시 문제 번호와 접근법, 입력/출력 가정, 주요 알고리즘 선택 이유를 간단히 적고, UI가 아닌 이상 스크린샷은 생략.

## README 문제 분류 표 작성 가이드
- 대상: README.md에 추가하는 “문제 분류 (유형별)” 표.
- 열 구성: `플랫폼 | 티어 | 문제번호 및 문제명 | 주요 도구/아이디어`.
- 문제번호/문제명 형식: `[<문제번호>. <문제명>](BOJ/.../<문제번호>/<문제번호>.md)`처럼 번호 뒤에 점(`.`)을 넣고, 레포 내 해당 문제 폴더의 마크다운 파일로 링크한다.
- “주요 도구/아이디어”에는 그 문제를 푸는 핵심 자료구조, 알고리즘, 혹은 해당 문제만의 독특한 아이디어를 한 줄로 적는다(풀이 요약이 아니라 핵심 키워드 위주).
- 유형별 섹션 헤더 예시: `### DP`, `### 그래프/최단거리`, `### 문자열/파싱` 등 문제 성격에 맞춰 배치한다.

## 복기용 마크다운 작성 가이드 (`<문제번호>.md`)
- `~/.template/recap_template.md`의 헤딩 흐름을 따라 사용: 제목 - 문제 정보 → 사고흐름 → 핵심아이디어 → 풀이(n) → 정리 요약.
- 양식 템플릿: `~/.template/recap_template.md`
- 모범샘플 : `BOJ/10000~/11286/11286.md`

- 그리고 아래는 구체적인 작성 가이드 라인이야
- 또한 n+1 번째 풀이로 당신의 모범풀이 코드도 보여줘
- AI가 만든 추가 코드가 있으면 `in.txt`가 존재할 때 이를 사용해 간단한 실행 결과를 확인하고, **기대와 다를 때만** 실행 명령과 기대/실측 출력(어디서 어긋났는지)을 복기 문서에 적어줘

# 코딩 테스트 복기(Markdown) 작성 가이드라인

본 문서는 사용자가 제공한 모범 샘플을 기반으로 AI 에이전트가 일관된 스타일의 코딩 테스트 복기 문서를 생성하기 위한 지침이다.

복기용 문서의 목적
1. 내 풀이에 대한 복기
2. AI모델이 생성한 모범답안 풀이 학습
3. 비슷한 문제나 다른 상황에서의 접근 방법
4. 언어 숙련도 향상을 위한 팁 획득

## 1. 문서 기본 구조 및 순서

모든 문서는 아래의 섹션 순서를 엄격히 준수한다.

1.  **문서 제목 (#)**
2.  **문제 정보 (##)**
3.  **사고흐름 (##)**
4.  **핵심아이디어 (##)**
5.  **풀이 (##)** (필요시 풀이 1, 풀이 2로 분리)
6.  **비슷한 문제에서의 사고과정 (##)**
7.  **정리 요약 (##)**

---

## 2. 섹션별 상세 작성 지침

### 1) 문서 제목
* 형식: `[문제 출처] [문제 번호] — [문제 이름] (w/ [사용 모델 버전])`
* 예시: `# BOJ 11286 — 절댓값 힙 (w/ gpt-5.1-codex)`
* 제목 밑에는 티어와 링크를 같이 넣어줘
* 예시: `(G4) <https://www.acmicpc.net/problem/14722>`

### 2) 문제 정보
* 문제의 핵심 요구사항과 연산 규칙을 요약한다.
* 입력 크기(N) 등 주요 제약 조건을 명시한다.

### 3) 사고흐름
* 문제를 보고 어떤 자료구조나 알고리즘을 떠올렸는지 단계별로 서술한다.
* 자료구조 선택의 근거를 명확히 한다. (예: 최솟값 반복 추출 -> 힙)
* 특이 조건(예: 절댓값 정렬, Tie-breaking)을 처리하기 위한 논리를 기술한다.
* 시간 복잡도 계산을 포함하여 해당 풀이가 유효함을 증명한다.

### 4) 핵심아이디어
* 문제를 풀기 위한 가장 중요한 기술적 포인트를 한 문단으로 요약한다.
* 구현의 핵심 원리를 굵은 글씨로 강조한다.
    * 예: **단일 힙에 튜플 형태로 저장하여 정렬 기준 인코딩**

### 5) 풀이 (풀이 1, 풀이 2 ...)
* **코드 블록:** 언어를 명시한 코드 블록을 사용한다.
* **핵심 포인트:** 코드에서 주의 깊게 봐야 할 구현 요소를 불릿 포인트로 나열한다.
* **로직 설명 (중요):**
    * 각 풀이마다 **전체 코드를 먼저** 보여주고,
    * 이후 핵심 로직을 설명할 때는 **설명과 함께 해당 부분 코드 블록을 바로 붙여서** 보여준다(설명만 하지 말 것).
    * 로직이 복잡하다면 **구체적인 입력 예시**를 통해 데이터의 흐름을 보여준다.
    * 또한 필요에 따라서 자료구조(힙, 스택, 배열 등)의 내부 상태 변화를 단계별로 추적(Trace)하여 시각적으로 이해하기 쉽게 설명한다.
* **기존 코드 복기:** 내 기존 코드를 첨삭할 때에는, 내 풀이는 그대로 보존하고 그 풀이에 대한 설명과 피드백을 해준다.
* **기존 코드 첨삭:** 풀이의 전반적 로직을 유지하되, AI가 읽기 좋게 변수명/입출력/미세 최적화를 **가볍게 보완**해도 된다(주요 알고리즘 변경 없이).
* **장단점:** 해당 풀이의 시간/공간 효율성 및 구현 난이도 측면에서의 장단점을 분석한다.
* 디렉토리 안에 또 다른 풀이 (<문제번호>_<num>) 가 있다면 그 풀이도 풀이 1 과 같은 양식으로 작성해준다.
* 맨 마지막 풀이로 당신이 만든 추가 풀이도 작성해서 추가해준다. (내 풀이와는 완전 별개로 AI 모범답안을 참고하기 위함임.)

### 6) 비슷한 문제에서의 사고과정
* 이 문제에서 얻은 지식을 다른 문제에 어떻게 적용할지 일반화한다.
* 특정 문제에 국한되지 않는 '패턴'을 정의한다.
    * 예: 복합 우선순위 기준 -> 튜플 비교 활용
* **자주 하는 실수/오해 포인트**도 함께 적어준다(경계 조건, 중간 상태 전파, 정렬/우선순위 착각 등 문제 풀다가 말리기 쉬운 패턴).

### 7) 정리 요약
* 전체 내용을 2~3줄로 요약한다.
* 기억해야 할 알고리즘 패턴과 핵심 키워드를 다시 한번 강조한다.

## 3. 포맷팅 및 스타일 규칙

* **강조:** 핵심 키워드, 알고리즘 명칭, 주요 변수는 `**굵게**` 표시한다.
* **구분선:** 각 상위 섹션(`##`)이 끝날 때마다 수평선(`---`)을 삽입하여 구획을 나눈다.
* **어조:** 설명은 명확하고 간결하게 작성하며, 불필요한 서술을 배제한다.
* **수식:** 복잡한 수식은 LaTeX를 사용하고, 간단한 변수명이나 숫자는 백틱(`` ` ``)이나 일반 텍스트를 사용한다.

수정사항
(17413_2.py)
뒤집는 작업이 있으므로 배열로 관리하다가 마지막에만 join 으로 문자열로 만드는게 좋아보임
=> 따라서 common_word(word_buf) 와 res 를 배열로 변경


상태
* is_tag == True : 태그 안 (<...>)
* is_tag == False : 태그 밖 (단어 구간 + 공백)

로직
- 문자열을 한 글자씩 읽는다
- '<'를 만나면
  - 지금까지 모아둔 임시배열을 뒤집어서 결과에 추가 (flush_word)
- 공백(' ')을 만나면
  - 임시배열 뒤집어서 결과에 추가
  - 결과에 공백도 추가
- 태그 모드이면
  - 결과에 즉시 추가
  - '>' 를 만나면 태그 모드 플래그 해제
- 태그 모드가 아닌 일반문자이면
  - 임시배열에 추가

- 마지막 임시배열 뒤집어서 결과에 추가

코멘트
```
1. 상태를 먼저 정의한다

예: 태그 안 / 태그 밖, 문자열 / 숫자 / 공백, 따옴표 안 / 밖 등

2. 버퍼와 경계 문자를 정한다

어떤 문자를 만나면 지금까지 모은 버퍼를 처리하고 비울지?

이 문제에선 <, 공백, 문자열 끝이 경계.

3. 상태별로만 생각한다

if state == TAG: → 태그 안에서 가능한 행동만 생각

else: → 태그 밖에서만 고려할 규칙만 생각

이렇게 나누면 조건이 섞여서 꼬이는 걸 방지할 수 있다.

4. 버퍼 플러시는 함수로 빼기

<든 공백이든, “단어 끝”일 때마다 하는 행동이 똑같다면 무조건 함수로.

실전에서 버그 줄이는 데 도움됨
```
